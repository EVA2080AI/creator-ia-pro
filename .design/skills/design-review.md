# /design-review

Crítica estructurada contra brief.

## Checklist

### Layout & Composition
- [ ] Grid consistente
- [ ] Jerarquía visual clara
- [ ] Espaciado sistemático
- [ ] Responsive correcto

### Typography
- [ ] Legible
- [ ] Jerarquía de pesos
- [ ] Límites de longitud de línea
- [ ] Contraste suficiente

### Color
- [ ] Tokens usados correctamente
- [ ] Modo oscuro/claro funciona
- [ ] Estados (hover, focus, disabled)

### Interacción
- [ ] Estados definidos
- [ ] Feedback claro
- [ ] Animaciones coherentes
- [ ] Keyboard accessible

### Contenido
- [ ] Copy apropiado
- [ ] Microcopy útil
- [ ] Formatos de fecha/hora consistentes

### Performance
- [ ] Imágenes optimizadas
- [ ] Sin layout shift
- [ ] Carga progresiva

## Uso

```
/design-review [feature]
```

## Output

Archivo: `.design/reviews/[feature]-[fecha].md`
