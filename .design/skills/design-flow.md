# /design-flow

Orquesta secuencia completa de workflow de diseño.

## Secuencia

1. **Discovery** — Entender problema, usuarios, contexto
2. **Information Architecture** — Navegación, jerarquía, flujos
3. **Design Brief** — Documento escrito con objetivos
4. **Design Tokens** — Definir sistema visual
5. **Frontend Design** — Implementación con filosofía
6. **Design Review** — Validar contra brief

## Uso

Ejecutar al inicio de cualquier feature nueva:
```
/design-flow [nombre-feature]
```

## Output

- Carpeta `.design/briefs/[feature]/`
- Documentación completa
- Tokens específicos si aplica
- Checklist de tareas
