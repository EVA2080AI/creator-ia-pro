# /design-brief

Brief estructurado con exploración de codebase.

## Estructura

```markdown
# Brief: [Feature Name]

## Contexto
- ¿Por qué ahora?
- ¿Qué problema resuelve?

## Usuarios
- Primary persona
- Casos de uso

## Objetivos
- Métricas de éxito
- Deadline

## Exploración de Codebase
- Componentes existentes que reutilizar
- CSS variables disponibles
- Patrones establecidos

## Requisitos
- Must have
- Should have
- Nice to have

## Referencias
- Competencia
- Inspiración
- Benchmarks

## Notas
- Restricciones técnicas
- Consideraciones especiales
```

## Uso

```
/design-brief [nombre-feature]
```

## Output

Archivo: `.design/briefs/[feature]/brief.md`
